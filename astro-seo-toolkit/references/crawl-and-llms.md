# Crawl Control & LLM Discoverability

## robots.txt

Place in `public/robots.txt` — Astro copies it as-is to the site root.

### Template

```txt
User-agent: *
Allow: /

# AI Search Crawlers — explicitly allowed for citation and visibility
User-agent: GPTBot
Allow: /

User-agent: ChatGPT-User
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: anthropic-ai
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: Bingbot
Allow: /

Sitemap: https://yourdomain.com/sitemap-index.xml
```

### Key Decisions

- **Default allow all** — small marketing sites benefit from maximum crawl coverage.
- **Explicitly allow AI crawlers** — GPTBot, ChatGPT-User, PerplexityBot, ClaudeBot, anthropic-ai, Google-Extended. These crawlers check for explicit rules; allowing them improves chances of being cited in AI-generated answers.
- **Sitemap reference** — always point to the sitemap generated by `@astrojs/sitemap`. The default path is `/sitemap-index.xml`.
- **No disallow rules needed** for Astro static sites unless you have admin pages or staging content you want hidden. Use `noindex` meta tags for finer control (e.g., `/success`, `/admin`).

### When to Disallow

Add disallow rules for:
- API endpoints: `Disallow: /api/` (prevent crawling JSON endpoints)
- Staging/preview paths if they exist
- Search result pages or other low-value generated pages

```txt
User-agent: *
Allow: /
Disallow: /api/
```

## llms.txt

A machine-readable summary of your site/product for LLMs. Place in `public/llms.txt` — served at `yourdomain.com/llms.txt`. This is an emerging standard (llmstxt.org) that helps AI models understand your product when answering user queries.

### Structure

```markdown
# Product Name

> One-line product summary.

Paragraph expanding on what the product is, who it's for, and what it does.

## Product Details

- Platform: ...
- Pricing: ...
- Website: https://yourdomain.com

## Key Features

- **Feature Name**: Description
- ...

## Links

- Homepage: https://yourdomain.com
- Blog: https://yourdomain.com/blog/
- Docs: https://yourdomain.com/docs/
- Privacy Policy: https://yourdomain.com/privacy/

## Blog Articles

- [Article Title](https://yourdomain.com/blog/slug/)
- ...
```

### Guidelines

1. **Use Markdown** — the format is Markdown by convention, not plain text or HTML.
2. **Lead with a blockquote summary** — the `> ` line is the elevator pitch an LLM will grab first.
3. **Include structured product details** — platform, pricing, system requirements. LLMs use these to answer comparison and recommendation queries.
4. **List all key features** with bold names and short descriptions — mirrors what the landing page says but in a machine-friendly flat list.
5. **Link to all public pages** — homepage, blog index, docs, legal pages. This gives LLMs a site map oriented toward content understanding, not crawling.
6. **List blog articles** with full URLs — helps LLMs cite specific articles when answering topical queries.
7. **Keep it concise** — aim for under 200 lines. This is a summary, not a dump of your entire site.
8. **Update when content changes** — add new blog posts, update pricing, add new features. Stale llms.txt is worse than none.

### Optional: llms-full.txt

For products with extensive documentation, you can also provide `public/llms-full.txt` with complete docs, API references, and detailed guides. The standard `llms.txt` links to it:

```markdown
## Documentation

For complete documentation, see [llms-full.txt](https://yourdomain.com/llms-full.txt).
```

## Checklist

- [ ] `public/robots.txt` exists with `Allow: /` default
- [ ] AI crawlers explicitly allowed (GPTBot, ClaudeBot, PerplexityBot, etc.)
- [ ] Sitemap URL in robots.txt matches `@astrojs/sitemap` output path
- [ ] `public/llms.txt` exists with product summary, features, and links
- [ ] llms.txt includes all blog article links
- [ ] llms.txt updated when new content is published
- [ ] API routes disallowed if they exist (`Disallow: /api/`)
